## 6) How we work (the sync ritual)
- **One source of truth per domain:** curriculum в†’ Curriculum Reference В· constitution в†’ Foundation В· design в†’ Design Brief В· pulse/status в†’ this file. Truth is not duplicated across two files.
- **Start of any session** (Code/Design): "read `START_HERE.md` first before you begin".
- **End of any Code session**: "update `START_HERE.md`: status + session log + any new decision".
- **A decision is not adopted until it is written in its domain's source of truth**, then referenced here. The direction: decision в†’ domain document в†’ pulse here (not the reverse silently).
- **When returning to Claude (Chat)**: share the latest version of this file.
- **(Methodological lesson вЂ” a migration ships with its code):** `db:migrate` hits the production database immediately; so any gate carrying a migration is **pushed and deployed in the same round** вЂ” no local unpushed commits left behind, otherwise a "database ahead of deployed code" window opens that breaks production. (Happened 2026-06-25 with 0062/0063.)
- **(Methodological lesson вЂ” every deploy ends with a live human check):** "green build" proves compilation, not live behavior; **the final verification is by eye on the deployed site**, not by a report alone.
- **(Methodological lesson вЂ” "build green" hides whole classes of faults В· confirmed 2026-06-27):** translation faults (raw i18n keys), font faults, and display/layout issues **do not break the build** вЂ” they surface only at runtime. Every affected surface needs a **live visual check** + confirming **Vercel Ready = HEAD**, not just a green compile.
- **(Methodological lesson вЂ” font bugs need both-direction visual checks):** verify *both* "Arabic not broken" *and* "English now correct". Rule-level / CSS-resolution proof can pass while the visual still fails вЂ” the `Nunito Fallback` (local Arial) Arabic interception was invisible to rule-level analysis and only showed by eye (2026-06-27).
- **(Methodological lesson вЂ” delete a file together with its consumer in the SAME commit):** a failed `git add` on a deleted-file pathspec silently splits the change and ships a broken build (missing-module). Use `git add -A` (or `git rm`) and check `git status` before committing.
- **(Methodological lesson вЂ” read deploy health from the TOP of the Deployments list):** the newest deployment is live; judging health from a stale failed deployment opened by name gives a false negative (2026-06-27).

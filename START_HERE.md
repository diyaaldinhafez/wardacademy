# نقطة البداية — أكاديمية وَرد (Single Source of Truth)

> هذا الملفّ هو **مصدر الحقيقة الوحيد** للمشروع. أيّ مساعدٍ (Claude Code / Claude Design / Claude في الدردشة) يجب أن **يقرأ هذا الملفّ أولاً** قبل أيّ عمل. **Claude Code مسؤولٌ عن تحديثه** في نهاية كل جلسة.

---

## ١) ما هو المشروع
أكاديمية وَرد — منصّة تعليم إنجليزية للأطفال **٩–١٣**، جلساتٌ فرديةٌ مباشرةٌ مع معلّم، يساعده ذكاءٌ اصطناعيّ **والمعلّم يعتمد كلّ مخرَجٍ قبل وصوله للطفل**. مساران: **منهج مدرسيّ** أو **مستويات CEFR**.

## ٢) الحالة الحالية
- ✅ **نظام التصميم** (Claude Design): مكتمل ومتماسك.
- ✅ **صفحة الهبوط**: مبنيةٌ ومنشورة (visual-only)، عربيةٌ افتراضيةٌ + RTL، فئة ٩–١٣، نموذج الثقة ظاهر.
- ✅ **الهوية واللغة**: البنفسجيّ بطل، الزهرة هوية/الشرارة ذكاء، الخطوط مضبوطة.
- ⏳ **الأساس (backend)**: لم يبدأ — **الخطوة التالية**.
- ❌ **جوهر المنتج** (مصادقة، تخزين، عزل بيانات، توليد الذكاء + بوابة اعتماد المعلّم): غير مبنيّ. الشاشات الحالية **قشورٌ بصرية** حتى يُبنى هذا.

## ٣) القرارات المثبّتة (لا يُعاد فتحها بلا سبب)
- الهوية **مرتكزةٌ على وليّ الأمر** (حسابٌ واحدٌ لوليّ الأمر، أبناءٌ متعدّدون، موافقةٌ شرطٌ لكل حساب).
- **المعلّم يعتمد كلّ مخرَج ذكاء** قبل وصوله للطفل (نموذج الثقة الأساسيّ).
- **التقدّم ثنائيّ المسار**: البتلات = مهارات تُملأ من **الأهداف المتقنة** (لا درجات، والإتمام منفصل)؛ bud→bloom = وحدة؛ النطاق التراكميّ: CEFR=مستوى/موسم، مدرسيّ=كتاب/فصل. أرقامٌ صادقة.
- **بيانات أوروبا + عزل RLS** بين الطلاب.
- **توليدٌ أصليّ** للمحتوى (لا ابتلاع كتبٍ محميّة).
- **الفئة ٩–١٣**، عربيٌّ للأهل/المعلّم وإنجليزيٌّ لسطح الطفل.
> التفاصيل الكاملة في: `PRD`, `Build Brief`, `Design System Brief`.

## ٥) الأدوار (حتى لا يتداخلوا)
- **Claude (الدردشة)** = الاستراتيجية، القرارات، المراجعة. (يحتفظ بذاكرةٍ عبر المحادثات — مرجع الاستمرارية.)
- **Claude Code** = البناء الفعليّ + **صيانة هذا الملفّ**.
- **Claude Design** = التصميم البصريّ على دفعات؛ مخرجاته تذهب للمستودع عبر الـ handoff.

## ٦) طريقة العمل (طقس المزامنة)
- **بداية أيّ جلسة** (Code/Design): «اقرأ `START_HERE.md` أولاً قبل أن تبدأ».
- **نهاية أيّ جلسة Code**: «حدّث `START_HERE.md`: ماذا تغيّر، وأيّ قرارٍ جديد».
- **عند العودة إلى Claude (الدردشة)**: شارِك آخر نسخةٍ من هذا الملفّ.

## ٧) الخطوة التالية
**المرحلة ٠ (تهيئة الخدمات بأمان):** سقف تكلفة Anthropic أولاً ← Supabase في أوروبا + RLS ← Sentry.
**ثم المرحلة ١ (الأساس):** مصادقة مُدارة، نموذج بيانات بوسم الأهداف، عزل RLS، خدمة توليد، **بوابة اعتماد المعلّم**، وشريحةٌ عمودية: هدفٌ ← توليد عنصرٍ أصليّ ← اعتماد المعلّم.

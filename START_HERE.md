# نقطة البداية — أكاديمية وَرد (Single Source of Truth)

**النسخة:** 3 · **آخر تحديث:** 2026-06-17 (حدّثه Claude Code)

> هذا الملفّ هو **مصدر الحقيقة الوحيد** للمشروع. أيّ مساعدٍ (Claude Code / Claude Design / Claude في الدردشة) يجب أن **يقرأ هذا الملفّ أولاً** قبل أيّ عمل. **Claude Code مسؤولٌ عن تحديثه** في نهاية كل جلسة.

## ١) ما هو المشروع
أكاديمية وَرد — منصّة تعليم إنجليزية للأطفال **٩–١٣**، جلساتٌ فرديةٌ مباشرةٌ مع معلّم، يساعده ذكاءٌ اصطناعيّ **والمعلّم يعتمد كلّ مخرَجٍ قبل وصوله للطفل**. مساران: **منهج مدرسيّ** أو **مستويات CEFR**.

## ٢) الحالة الحالية
**الواجهة التسويقية مبنيّةٌ بصرياً، والأساس (backend) + مساحات العمل مبنيّةٌ وعاملةٌ ومُختبَرةٌ بعزلٍ حقيقي، ومنشورةٌ حيّاً على ward.academy (Vercel/fra1).**

- ✅ **نظام التصميم** (Claude Design): مكتمل ومتماسك.
- ✅ **صفحة الهبوط**: منشورة على ward.academy، عربيةٌ افتراضيةٌ + RTL، فئة ٩–١٣. أُعيد بناؤها لتكون **مقودةً بالمنتج (تُري لا تُخبر)** بالأقسام: البطل (وردة تتفتّح عند الظهور + لمحة تقرير التفتّح) · كيف نعمل · إطلالة من الداخل (محاكاة تطبيق وليّ الأمر + الطالب) · تقرير التفتّح (الوردة = خريطة مهارات) · المعلّم الخاص (جلسة فردية) · دعم الذكاء الاصطناعي (محاكاة لوحة المعلّم: «لا يصل شيء للطالب قبل موافقتك») · النتائج (أرقام نموذجية) · الأسعار (٣ باقات، ميزات مشتركة) · آراء الأهل · الأسئلة الشائعة · الدعوة الأخيرة. مع: ظهور تدريجي عند التمرير، زرّ حجز ثابت على الجوّال، خلفية بطل ممتدّة، أزرار حجز خضراء، وصورة OG.
- ✅ **الصفحات الفرعية (محاكاة)**: `/enroll` (تسجيل ← حجز موعد ← اختبار مستوى، مع تحقّق من النموذج) · `/login` (دخول مرئيّ) · `/placement` (اختبار + نتيجة «بانتظار اعتماد المعلّم»).
- ✅ **الهوية واللغة**: البنفسجيّ بطل، الزهرة هوية/الشرارة ذكاء، الخطوط مضبوطة، **أرقام غربية**، **بلا إيموجي**، **بلا أسماء كاملة** (أسماء أولى فقط في المحاكاة)، خطاب الذكاء الاصطناعي **مخفَّف للأهل** (التفاصيل للمعلّم).
- 🗑️ **مُزال**: صفحتا `/teacher` و`/brand` (لوحة المعلّم بُنيت حقيقيةً في `/studio`).
- ✅ **الأساس (backend) — مبنيّ ومُختبَر (١٢ هجرة على Supabase/Frankfurt، RLS على كل جدول):**
  - **العزل والهوية:** `tenants` (وحدة العزل) · `profiles` (هوية + أدوارٌ بالطبقات) · خطّاف JWT يحقن `tenant_id`+`roles` (مفعّل). لا تصعيد أدوار (لا كتابة عميلٍ على `profiles`).
  - **أولياء الأمور والطلاب:** `guardianships` (واحد→متعدّد + موافقةٌ لكل ابن) · رؤية الملفّات مقيّدةٌ بالدور (وليّ الأمر يرى أبناءه فقط، المعلّم يرى مستأجِره).
  - **محرّك المحتوى:** `objectives` (CEFR/مدرسي) · `items` (موسومةٌ بالهدف) + **بوّابة اعتماد المعلّم** · `item_keys` (الإجابة محجوبةٌ عن الطالب) · `assignments` (إسنادٌ لكل طالب).
  - **الإرسالات والتقدّم:** `submissions` (تصحيحٌ موثوقٌ على الخادم، لا تزوير) · `progress_records` (عدّاتٌ صادقة، تحديثٌ متزامنٌ بـ trigger، الإتمام منفصلٌ عن الأداء).
  - **الجلسات:** `sessions` (UTC + منع تعارضٍ على مستوى DB) · `session_reports` (مسوّدةٌ بالذكاء → تحرير → اعتماد → تظهر للعائلة).
  - **خدمة توليدٍ واحدة** (`lib/generation/service.ts`، Claude) للعناصر الأصلية ومسوّدات التقارير.
- ✅ **مساحات العمل (Next.js، إنجليزي/LTR، محميّةٌ بـ proxy):** `/studio` (المعلّم: توليد/اعتماد/إسناد/جدولة/تقارير + تقدّم الطلاب) · `/learn` (الطالب: حلّ المُسنَد + تقدّمه + جلساته) · `/guardian` (وليّ الأمر: أبناؤه + موافقة + تقدّم + تقارير) · `/signup` (تسجيلٌ ذاتيّ) · دخولٌ موجَّهٌ حسب الدور.
- ✅ **أدوات + بذرة:** `npm run db:migrate` / `db:status` / `db:seed`. حسابات تجريبية: `teacher@ward.local` / `student@ward.local` / `parent@ward.local`.

## ٣) القرارات المثبّتة (لا يُعاد فتحها بلا سبب)
- الهوية **مرتكزةٌ على وليّ الأمر** (حسابٌ واحدٌ لوليّ الأمر، أبناءٌ متعدّدون، موافقةٌ شرطٌ لكل حساب).
- **المعلّم يعتمد كلّ مخرَج ذكاء** قبل وصوله للطفل (نموذج الثقة الأساسيّ).
- **التقدّم ثنائيّ المسار**: البتلات = مهارات تُملأ من **الأهداف المتقنة** (لا درجات، والإتمام منفصل)؛ bud→bloom = وحدة؛ النطاق التراكميّ: CEFR=مستوى/موسم، مدرسيّ=كتاب/فصل. أرقامٌ صادقة.
- **بيانات أوروبا + عزل RLS** بين الطلاب.
- **توليدٌ أصليّ** للمحتوى (لا ابتلاع كتبٍ محميّة).
- **الفئة ٩–١٣**، عربيٌّ للأهل/المعلّم وإنجليزيٌّ لسطح الطفل.
- **في الواجهة للأهل:** الذكاء الاصطناعي «داعمٌ للمعلّم والطالب» لا أكثر؛ تفاصيله التقنية تبقى في واجهة المعلّم. أرقام غربية، بلا إيموجي، أسماء أولى فقط.
> التفاصيل الكاملة في: `PRD`, `Build Brief`, `Design System Brief`.

## ٨) الحقائق التقنية المثبّتة
- **الإطار:** Next.js 16 (App Router) + React 19 + TypeScript + Tailwind v4 + Turbopack.
- **Node 20 إلزاميّ** (Node 24 غير متوافق مع edge-runtime في Next 16) — مثبّت في `.nvmrc`.
- **النشر:** Vercel على `ward.academy`. **المستودع:** github.com/diyaaldinhafez/wardacademy، الفرع `main`.
- **اللغة:** `LanguageProvider` (AR افتراضي/RTL ↔ EN) للتسويق؛ مساحات العمل إنجليزيةٌ/LTR.
- **القاعدة:** Supabase (PostgreSQL، EU/Frankfurt) + RLS. الهجرات `supabase/migrations/*.sql` تُطبَّق بـ `npm run db:migrate` (سكربت pg، سلسلة Session pooler في `.env.local` كـ `SUPABASE_DB_URL`).
- **عملاء Supabase:** `lib/supabase/{client,server,admin}.ts` + `proxy.ts`/`lib/supabase/session.ts` (تحديث الجلسة وحماية `/studio` `/learn` `/guardian`).
- **الذكاء:** `lib/generation/service.ts` (server-only) عبر `@anthropic-ai/sdk` (نموذج `claude-sonnet-4-6`)؛ سقف إنفاق Anthropic مضبوط ($5 تجريبي).
- **التحقّق:** `npm run build` قبل أي رفع. `.env.local` خارج Git (Supabase + Anthropic + DB URL).

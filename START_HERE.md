## 6) How we work (the sync ritual)
- **One source of truth per domain:** curriculum вЖТ Curriculum Reference ¬Ј constitution вЖТ Foundation ¬Ј design вЖТ Design Brief ¬Ј pulse/status вЖТ this file. Truth is not duplicated across two files.
- **Start of any session** (Code/Design): "read `START_HERE.md` first before you begin".
- **End of any Code session**: "update `START_HERE.md`: status + session log + any new decision".
- **A decision is not adopted until it is written in its domain's source of truth**, then referenced here. The direction: decision вЖТ domain document вЖТ pulse here (not the reverse silently).
- **When returning to Claude (Chat)**: share the latest version of this file.
- **(Methodological lesson вАФ a migration ships with its code):** `db:migrate` hits the production database immediately; so any gate carrying a migration is **pushed and deployed in the same round** вАФ no local unpushed commits left behind, otherwise a "database ahead of deployed code" window opens that breaks production. (Happened 2026-06-25 with 0062/0063.)
- **(Methodological lesson вАФ every deploy ends with a live human check):** "green build" proves compilation, not live behavior; **the final verification is by eye on the deployed site**, not by a report alone.
- **(Methodological lesson вАФ "build green" hides whole classes of faults ¬Ј confirmed 2026-06-27):** translation faults (raw i18n keys), font faults, and display/layout issues **do not break the build** вАФ they surface only at runtime. Every affected surface needs a **live visual check** + confirming **Vercel Ready = HEAD**, not just a green compile.
- **(Methodological lesson вАФ font bugs need both-direction visual checks):** verify *both* "Arabic not broken" *and* "English now correct". Rule-level / CSS-resolution proof can pass while the visual still fails вАФ the `Nunito Fallback` (local Arial) Arabic interception was invisible to rule-level analysis and only showed by eye (2026-06-27).
- **(Methodological lesson вАФ delete a file together with its consumer in the SAME commit):** a failed `git add` on a deleted-file pathspec silently splits the change and ships a broken build (missing-module). Use `git add -A` (or `git rm`) and check `git status` before committing.
- **(Methodological lesson вАФ read deploy health from the TOP of the Deployments list):** the newest deployment is live; judging health from a stale failed deployment opened by name gives a false negative (2026-06-27).
